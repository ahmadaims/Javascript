JavaScript Strings

Strings are for storing text
Strings are written with quotes

Using Quotes

A JavaScript string is zero or more characters written inside quotes.

let clientName = "John Doe";
You can use single or double quotes: 
let carName1 = "Volvo XC60";  // Double quotes
let carName2 = 'Volvo XC60';  // Single quotes

Quotes Inside Quotes

You can use quotes inside a string, as long as they don't match the quotes surrounding the string

let answer1 = "It's alright";
let answer2 = "He is called 'Johnny'";
let answer3 = 'He is called "Johnny"';

Below double quotes inside double quotes will generate error.

let answer3 = "He is called "Johnny"";

Template Strings

  • Templates were introduced with ES6 (JavaScript 2016).
  • Templates are strings enclosed in backticks (`This is a template string`).
  • Templates allow single and double quotes inside a string:

Example

let text = `He's often called "Johnny"`;

Note

Templates are not supported in Internet Explorer.

String Length

To find the length of a string, use the built-in length property:

let abc = "ABCDEFGHIJKLMNOPQRSTUVWXYZ";
let length = abc.length;

JS Strings Escape Characters

Escape characters in JavaScript strings are special sequences used to represent characters that would otherwise have a special meaning within the string itself, or to insert non-printable characters. Here's a breakdown:

Why Escape Characters?
  • Double quotes (") and single quotes (') are commonly used to delimit strings in JavaScript.
  • However, if you want to include a quote within a string itself (e.g., "This is a 'cool' feature"), there would be a conflict.
  • Escape characters help you include these special characters literally within the string.

1. Including Quotes Within Strings:

The most common use case is to include quotes (single or double) within a string that uses the same type of quote for delimiting. Escape characters allow you to represent the quote literally within the string.

Code:

const quoteinsidequote = "This is a quote \"inside\" the string.";

console.log(quoteinsidequote);

2. Formatting Text with Newlines and Tabs:

Use escape characters to create multi-line strings or formatted text with tabs.

Code of newline by using (\n) and column by using (\t):

let poem = "Roses are red,\nViolets are blue,\nJavaScript is awesome,\nand so are you!";
console.log(poem);
poem = "Roses are red,\tViolets are blue,\tJavaScript is awesome,\tand so are you!";
console.log(poem);

In most cases, it's better to use template literals over single or double quotes for strings in modern JavaScript.Here's why:

Benefits of Template Literals:

  • Readability: Template literals (backticks) allow you to embed expressions directly within strings, making code easier to read and understand. No more mental gymnastics to figure out single vs. double quotes for escaping.
  • String Interpolation: You can seamlessly include variables and expressions within strings using ${expression} syntax. This eliminates the need for string concatenation (+) for dynamic string creation. 
    const name = "Alice";
const greeting = `Hello, ${name}!`; // Clearer than "Hello, " + name + "!"
  • Multiline Strings: You can write multiline strings without the need for concatenation or escape characters for newlines. 
    const message = `This is a multi-line string
that spans across multiple lines.`;
